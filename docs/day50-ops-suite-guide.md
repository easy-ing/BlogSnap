# Day50 Ops Suite Guide

## 목적
- 운영 점검 자동화 흐름(Day45~Day49)을 한 번에 실행한다.

## 실행 단계
1. Day45 release health bundle
2. Day46 alert noise review
3. Day47 incident watcher
4. Day48 deploy approval gate
5. Day49 retro autofill

## 출력
- `tmp/reports/day50-ops-suite-<timestamp>.md`
- `tmp/reports/day50-ops-suite-<timestamp>.json`

## 사용 시점
- 배포 이후 일일 운영 체크
- 주간 운영 품질 점검 전 사전 실행
